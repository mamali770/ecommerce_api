## Overview
این پروژه یک API فروشگاهی است که با Laravel 12 توسعه داده شده و تمرکز آن روی پیاده‌سازی اصولی Backend، ساختار تمیز و الگوهای واقعی بازار کار است.  
این API برای استفاده در وب‌سایت یا اپلیکیشن موبایل طراحی شده است.

---

## Tech Stack
هسته پروژه با Laravel 12 و PHP پیاده‌سازی شده و از MySQL به‌عنوان دیتابیس استفاده می‌کند.  
احراز هویت کاربران به‌صورت توکنی و با Laravel Sanctum انجام شده است.

---

## Authentication
سیستم احراز هویت با Sanctum پیاده‌سازی شده و شامل ثبت‌نام، ورود و محافظت از routeها است.  
ساختار پروژه به‌گونه‌ای است که در صورت نیاز، امکان توسعه role و permission وجود دارد.

---

## Domain Models
مدل‌های اصلی پروژه شامل موارد زیر هستند:

- User  
- Province  
- City  
- Brand  
- Category  
- Product  
- ProductImage  
- Order  
- OrderItem  
- Transaction  

روابط بین مدل‌ها مطابق سناریوهای واقعی فروشگاهی طراحی شده‌اند.

---

## Core Features

### Products
مدیریت کامل محصولات شامل ایجاد، ویرایش، حذف و دریافت لیست.  
هر محصول به برند و دسته‌بندی متصل است و امکان ثبت چند تصویر برای آن وجود دارد.

### Orders
کاربران می‌توانند سفارش ثبت کنند و هر سفارش شامل چند آیتم است.  
اطلاعات محصول به‌صورت snapshot ذخیره می‌شود تا تاریخچه سفارش‌ها ثابت بماند.

### Transactions
تراکنش‌ها به سفارش متصل هستند و ساختار پروژه برای اتصال به درگاه پرداخت واقعی آماده است.

---

## API Response Structure
تمام پاسخ‌های API از یک ساختار یکسان پیروی می‌کنند:

```json
{
  "message": "Request was successful",
  "data": {}
}
```

این ساختار باعث می‌شود مدیریت پاسخ‌ها در سمت کلاینت ساده و قابل پیش‌بینی باشد.

## Validation and Error Handling
اعتبارسنجی ورودی‌ها با Form Request انجام شده و خطاها به‌صورت متمرکز مدیریت می‌شوند.
برای هر وضعیت، HTTP Status Code مناسب بازگردانده می‌شود.

## Notes
این پروژه با رویکرد Backend محور توسعه داده شده و به‌عنوان نمونه کد برای موقعیت‌های شغلی مرتبط با Laravel API قابل استفاده است.
